# Feature Reference

This document describes all columns present in the processed credit sales dataset used for model training. Anyone replicating this study must supply institutional data that can be transformed into this schema via the `CreditSalesProcessor` pipeline (`src/modules/feature_engineering/credit_sales_machine_learning.py`).

---

## Index / Identity Columns

These columns uniquely identify each invoice record and are not used as model inputs.

| Column | Description |
|---|---|
| `school_year` | Academic year of the invoice (e.g., `2023-2024`) |
| `student_id_pseudonymized` | Anonymized student identifier |
| `category_name` | Fee category (e.g., tuition, miscellaneous) |

---

## Raw Financial Columns

Derived directly from the source revenue records.

| Column | Description |
|---|---|
| `gross_receivables` | Total billed amount before deductions |
| `amount_discounted` | Discount applied to the invoice |
| `adjustments` | Post-billing adjustments (credit or debit) |
| `credit_sale_amount` | Net receivable after discount and adjustments |
| `due_date` | Contractual payment due date |
| `date_fully_paid` | Date on which the invoice was fully settled (NaT if not yet paid) |

---

## Days-to-Payment (DTP) Features

Capture a student's historical payment speed across prior invoices.

| Column | Description |
|---|---|
| `dtp_1` | Days to full payment for the most recent prior invoice |
| `dtp_2` | Days to full payment for the 2nd most recent prior invoice |
| `dtp_3` | Days to full payment for the 3rd most recent prior invoice |
| `dtp_4` | Days to full payment for the 4th most recent prior invoice |
| `dtp_avg` | Simple average of available DTP values |
| `dtp_wavg` | Weighted average of DTP values (more recent = higher weight) |
| `dtp_2_trend` | Slope of DTP across the 2 most recent invoices (negative = improving) |
| `dtp_3_trend` | Slope of DTP across the 3 most recent invoices |
| `days_since_last_payment` | Calendar days elapsed since the student's last full payment |
| `dtp_rolling_std` | Rolling standard deviation of recent DTP values |
| `dtp_max` | Maximum DTP value observed across recent invoices |

---

## Financial / Cumulative Features

Reflect the student's running financial position at the time of invoice issuance.

| Column | Description |
|---|---|
| `amount_due_cumsum` | Cumulative total billed to the student across all invoices to date |
| `amount_paid_cumsum` | Cumulative total paid by the student across all invoices to date |
| `opening_balance` | Outstanding unpaid balance carried over from prior periods |
| `opening_balance_flag` | Binary flag: 1 if the student has a non-zero opening balance |
| `payment_ratio` | Ratio of cumulative paid to cumulative billed (`amount_paid_cumsum / amount_due_cumsum`) |

---

## Behavioral / Historical Features

Encode patterns of payment behavior over the student's history.

| Column | Description |
|---|---|
| `prev_bracket` | Payment bracket of the student's most recent prior invoice |
| `early_payer_flag` | Binary flag: 1 if the student has a history of paying before the due date |
| `on_time_streak` | Count of consecutive on-time payments immediately preceding the current invoice |

---

## Payment Plan Features

Payment plan type is one-hot encoded. Each student is assigned one plan per invoice period.

| Column | Plan Description |
|---|---|
| `plan_type_Plan - A` | Full payment upfront |
| `plan_type_Plan - B` | Installment plan with up to 3 payments (1–30 days) |
| `plan_type_Plan - C` | Installment plan with up to 6 payments (31–60 days) |
| `plan_type_Plan - D` | Monthly installment plan (61+ days) |
| `plan_type_Plan - E` | Special COVID-era plan (applied to one school year only) |
| `plan_type_nan` | No plan assigned; student is not enrolled |
| `plan_type_risk_score` | Ordinal encoding of plan type by associated payment risk (A=0, B=1, C=2, D=3, E=4, NaN=5) |

---

## Temporal Features

Capture seasonal and cyclical effects on payment behavior.

| Column | Description |
|---|---|
| `due_month` | Calendar month of the due date (1–12) |
| `due_quarter` | Fiscal quarter of the due date (1–4) |

---

## Survival Analysis Features

Generated by fitting a **Cox Proportional Hazards model** (via `lifelines`) on censored training-set rows. The fitted model is then applied to both train and test splits to prevent data leakage. These features represent time-dependent hazard information.

| Column | Description |
|---|---|
| `partial_hazard` | Exponentiated linear predictor from the Cox model |
| `log_partial_hazard` | Log of the partial hazard |
| `expected_survival_time` | Expected days to full payment under the Cox model |
| `survival_probability` | Probability of remaining unpaid beyond a reference time point |
| `cumulative_hazard` | Cumulative hazard at the reference time point |

These features constitute the **"enhanced" feature phase**. The **"baseline" phase** uses all features above except these five.

---

## Target Variable

| Column | Description |
|---|---|
| `dtp_bracket` | Payment bracket label; one of `on_time`, `30_days`, `60_days`, `90_days` |
| `censor` | Survival analysis censoring flag: 0 = fully paid (event observed), 1 = censored (not yet paid as of the observation date) |
